# 🚀 Ansible + AWS Configuration Management Project

## 🔹 Project Summary

Automated the deployment and management of Nginx on multiple AWS EC2 instances using Ansible from a Ubuntu environment, demonstrating real-world configuration management practices.

---

## 🧪 What I Did

* Set up Ansible on  Ubuntu (Control Node)
* Launched and configured AWS EC2 instances (Ubuntu)
* Established secure SSH connectivity using key-based authentication
* Created an inventory file to manage multiple servers
* Verified connectivity using Ansible ping module
* Wrote an Ansible playbook to install and start Nginx
* Executed playbook to configure multiple servers simultaneously
* Accessed deployed application via browser

---

## ⚠️ Challenges Faced

* Faced SSH connectivity issues due to incorrect key permissions
* Encountered inventory parsing and configuration errors
* Nginx was not accessible initially due to closed port 80


---

## 🧠 What I Learned

* Fundamentals of Configuration Management
* Difference between manual vs automated server management
* Working with Ansible inventory and playbooks
* Importance of SSH and secure key handling
* Debugging real-world DevOps issues (SSH, inventory, security groups)
* Idempotency concept in Ansible
* Managing multiple servers with a single command

---

## 🚀 Why This Project Matters

This project demonstrates practical DevOps skills by automating server configuration in a cloud environment. It reflects the ability to work with real infrastructure, troubleshoot issues, and implement scalable automation — key skills required in modern DevOps roles.
